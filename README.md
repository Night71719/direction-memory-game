# 方向记忆联机小游戏

这是一个可在手机和电脑浏览器中多人联机游玩的方向记忆复原游戏。后端使用 Node.js、Express 和 Socket.IO，服务器负责房间、方向序列、轮次、倒计时、玩家状态和分数同步。

## 环境要求

- Node.js 18 或更新版本
- npm

## 安装依赖

```bash
npm install
```

## 启动服务器

Windows 用户也可以直接双击项目文件夹里的 `启动游戏.cmd`，它会启动服务器并打开浏览器。

```bash
npm start
```

服务器默认监听 `3000` 端口。

电脑本机访问：

```text
http://localhost:3000
```

## 手机通过同一 Wi-Fi 加入

1. 确保手机和运行服务器的电脑连接到同一个 Wi-Fi。
2. 在电脑上查看局域网 IP。
   - Windows 可以在终端运行：`ipconfig`
   - 找到类似 `192.168.x.x` 的 IPv4 地址。
3. 手机浏览器访问：

```text
http://电脑局域网IP:3000
```

例如：

```text
http://192.168.1.23:3000
```

如果手机无法打开，请检查电脑防火墙是否允许 Node.js 或 3000 端口被局域网访问。

## 手机直接点链接游玩

如果不想输入电脑局域网 IP，可以把项目部署到公网。部署后，手机直接打开公网网址即可创建房间或加入房间，手机也可以成为游戏房主。微信扫码本质上也是打开这个公网网址，不需要下载安装 App。

项目已包含：

- `render.yaml`
- `Procfile`
- `手机公网玩法说明.md`
- `微信扫码即玩说明.md`

这些文件用于更方便地部署到 Render、Railway、Fly.io、VPS 等支持 Node.js 的平台。

## 创建和加入房间

1. 玩家输入昵称。
2. 点击“创建房间”的玩家会自动成为房主。
3. 房主可以把房间号发给其他玩家，也可以让其他玩家直接扫描房间二维码。
4. 其他玩家输入昵称和房间号，或扫码后输入昵称，点击“加入”。
5. 房主可以设置总轮数，默认 10 轮，范围 1 到 30 轮。
6. 房主点击“开始游戏”后，所有房间内玩家同步开始。

游戏已经开始后，新玩家可以进入房间观战，但不会加入当前正在进行的回合；系统会在下一轮自动把他们加入正式玩家列表。

## 微信扫码加入

创建房间后，房间页会显示二维码和邀请链接。部署到公网 HTTPS 后，手机用微信扫一扫即可打开游戏并自动填入房间号。

- 同一 Wi-Fi 本地游玩时，二维码里的地址取决于当前浏览器地址。如果电脑用 `localhost` 打开，手机不能直接扫这个二维码加入，因为手机上的 `localhost` 指的是手机自己。
- 局域网手机加入时，建议电脑先用 `http://电脑局域网IP:3000` 打开游戏，再创建房间，这样二维码也会包含局域网 IP。
- 公网部署后，二维码会直接包含公网网址，微信扫码即可加入。
- 微信里最稳的是 HTTPS 公网网址，例如 `https://你的游戏域名.com`。

## 游戏规则

- 每轮服务器为整间房生成相同的随机序列，每个记忆项包含“箭头方向”和“显示颜色”。
- 奇数轮规则是“按颜色点击”：看箭头的颜色，点击相同颜色的按钮，不要管箭头朝向。
- 偶数轮规则是“按箭头点击”：看箭头方向，点击相同方向的按钮，不要管箭头颜色。
- 展示结束后，在规定时间内用屏幕按钮或键盘方向键按本轮规则复原。
- 完整输入后才判断对错。
- 全对加 1 分，任意错误或超时扣 1 分。
- 玩家完成后会等待其他玩家。
- 所有人完成或超时后，服务器统一进入下一轮。
- 房主离开后，房主权限会自动交给最早加入的其他玩家。
- 游戏结束后显示最终排行榜，房主可以点击“再来一局”。

## 键盘和触屏

- 手机：点击屏幕上的 `↑ ↓ ← →` 四个大按钮。
- 电脑：可以点击按钮，也可以按键盘方向键。
- 答题阶段以外按钮会自动禁用。

## 公网部署

可以部署到支持 Node.js 的平台，例如 Render、Railway、Fly.io、VPS 或云服务器。

通用步骤：

1. 上传整个项目。
2. 在服务器上运行 `npm install`。
3. 设置启动命令为 `npm start`。
4. 确保平台或服务器开放 Web 端口。
5. 如果平台提供 `PORT` 环境变量，本项目会自动使用该端口。

部署到公网后，玩家直接访问公网域名或扫码，即可通过房间号加入同一局游戏。
